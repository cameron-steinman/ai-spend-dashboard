ℹ Data Sources & Pricing

How Token Data is Collected

Every API call from Anthropic and Google logs a usage object in session JSONL files at ~/.openclaw/agents/main/sessions/*.jsonl. Each assistant message includes input, output, cacheRead, and cacheWrite token counts, plus a pre-calculated cost object from OpenClaw's billing layer. These are ingested into a local SQLite database by ingest-usage.py.

Perplexity (web_search): Tool calls don't emit token data. Cost is estimated at $0.005/call based on Sonar Pro pricing (~1K input + 500 output tokens typical). ESTIMATED

Groq, Ollama, OpenClaw internal: Calls logged but cost = $0 (free tiers or internal routing). Token counts from Groq/Ollama are often zero in the log.

Cost Calculation

Costs for Anthropic and Google are calculated by OpenClaw's billing layer at response time using actual token counts × published rates, then stored in the session JSONL. This dashboard reads those pre-calculated values directly — no re-calculation happens here.

Cache read tokens (Anthropic prompt caching) are billed at ~10% of input rate and tracked separately. Google implicit caching (auto since May 2025) reduces billed tokens on repeated prefixes — reflected in actual billed amounts.
